# Task 6 – Exception Handling & Debugging

## Objective
Learn how to handle runtime errors gracefully using Python exception handling.

## Concepts Covered
- `try`, `except`, `else`, `finally`
- Built-in exceptions (`ValueError`, `ZeroDivisionError`)
- Custom exceptions
- Error handling best practices

## Files Included
- `Task6.py`

## Program Features
- Accepts two numeric inputs
- Performs division safely
- Handles invalid inputs
- Prevents division by zero
- Raises custom exception for negative numbers

## Sample Cases
- Valid input → Displays result
- Zero division → Shows error message
- Non-numeric input → Shows validation error
- Negative input → Raises custom exception

## Interview Questions
### What is an Exception?
An exception is an error that occurs during program execution.

### Difference Between `try` and `except`
- `try`: Contains code that may cause an error
- `except`: Handles the error if it occurs

## Final Outcome
- Robust and error-free Python script
- Better debugging and exception handling skills
